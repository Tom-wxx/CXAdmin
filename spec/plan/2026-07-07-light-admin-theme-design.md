# 浅色后台整体风格设计说明

## 设计读法

这是一个面向高频操作人员的企业后台管理系统。目标不是做营销页式视觉冲击，而是在现有 Vue 3 + Element Plus 基础上，把整体风格调整为更稳定、清晰、耐看的浅色后台界面。

设计方向采用 **Quiet Light Admin**：

- 主题：全局浅色，不引入暗色区块作为主视觉。
- 气质：克制、清爽、低噪音，适合长时间处理表格、表单和监控页面。
- 设计系统：沿用 Element Plus，不额外引入新的 UI 库。
- 主色：保留当前青色品牌感，但降低大面积渐变和强动效的存在感。

## 现状审计

### 技术入口

- `frontend/src/styles/index.scss`：全局基础样式和 Element Plus 覆盖。
- `frontend/src/styles/refresh.scss`：当前“清新风”覆盖层，包含卡片、表格、导航、弹窗和动效。
- `frontend/src/layout/index.vue`：侧栏、顶部栏、主题设置入口和布局模式。
- `frontend/src/layout/components/AppMain.vue`：页面主内容背景与间距。
- `frontend/src/store/modules/settings.js`：侧栏默认颜色和布局模式。
- `frontend/src/components/ThemeSettings/index.vue`：主题预设。
- `frontend/src/views/index.vue`：首页 dashboard 的局部视觉。

### 当前问题

- 默认侧栏仍是深色，和“浅色整体风格”目标不一致。
- `refresh.scss` 中阴影、hover 位移、旋转动效较多，演示感偏强。
- 全局圆角存在 4px、10px、12px、14px、16px 混用，视觉规则不够统一。
- 首页 hero 使用大面积青色渐变，与后台日常操作界面密度不匹配。
- 表格、搜索区、工具栏已经有全局包装基础，适合做低风险整体调整。

### 保留内容

- 不调整路由、菜单结构、接口、权限逻辑。
- 不替换 Element Plus。
- 保留当前青色作为系统主色。
- 保留左侧布局和顶部布局两种模式。
- 保留现有业务页面结构，只改全局视觉和少量首页样式。

## 方案选择

### 推荐方案：Quiet Light Admin

将系统统一为浅色专业后台：

- 侧栏默认改为白色或极浅冷灰。
- 顶部栏保持白色，使用细边框和轻微阴影区分层级。
- 主内容区使用浅冷灰背景。
- 搜索表单、工具栏、表格、分页、弹窗保持白底。
- 卡片半径统一到 8px，按钮和输入框使用 Element Plus 默认半径或轻度覆盖。
- 动效只保留必要反馈，例如按钮 hover、表格行 hover、弹窗层级，不做大幅位移和旋转。

这个方案最适合后台管理系统，因为它优先保证扫描效率、信息稳定性和长期使用舒适度。

### 未采用方案

- Light Tech Blue：科技感更强，但浅蓝侧栏和蓝色强调容易抢业务内容注意力。
- Fresh Dashboard：改动最少，但保留较多演示型渐变和动效，整体不够沉稳。

## 视觉规则

### 色彩

- 页面背景：冷灰浅底，避免纯白页面直接铺满。
- 内容面板：白色。
- 主文本：近黑灰。
- 次级文本：中性灰。
- 边框：低饱和浅灰。
- 强调色：青色，仅用于主按钮、选中态、焦点态、关键图表。

### 形状

- 页面卡片、表格容器、弹窗：8px。
- 小控件内部圆角沿用 Element Plus，不额外做夸张圆角。
- 头像保持圆形。

### 阴影

- 默认尽量使用边框和留白表达层级。
- 只有浮层、下拉、弹窗使用更明显阴影。
- 常规卡片不做明显 hover 浮起。

### 动效

- 保留 150ms 到 220ms 的轻量状态过渡。
- 去掉全局旋转、明显上浮、强 stagger 入场。
- 遵守后台操作稳定性，避免布局移动。

## 实施范围

### 全局样式

修改 `frontend/src/styles/index.scss` 和 `frontend/src/styles/refresh.scss`：

- 统一浅色 token。
- 统一 Element Plus 主色覆盖。
- 调整表格头、行 hover、分页、标签、弹窗、下拉、加载遮罩。
- 降低阴影和动效强度。
- 修正部分旧注释乱码时顺手改为清晰中文注释。

### 布局壳

修改 `frontend/src/layout/index.vue`：

- 侧栏浅色化。
- 顶部栏和子顶部栏统一浅色边框规则。
- 菜单选中态改为浅色底 + 青色文字 + 左侧细条。
- 顶部模式下也遵循浅色主题。

### 主题设置

修改 `frontend/src/store/modules/settings.js` 和 `frontend/src/components/ThemeSettings/index.vue`：

- 默认侧栏颜色改为浅色。
- 主题预设以浅色为主，深色仅保留为可选项或移到后面。
- 重置主题恢复浅色默认值。

### 首页

修改 `frontend/src/views/index.vue`：

- 大面积渐变 hero 改为白色欢迎面板或轻量提示栏。
- 保留统计卡、图表、快捷入口，但统一 8px 半径、浅边框和低动效。
- 图表颜色继续使用青色为主，辅助色降低饱和度。

## 不做的事

- 不新增 UI 依赖。
- 不重写业务页面。
- 不调整后端。
- 不改变菜单数据、权限、接口路径。
- 不做暗色模式。
- 不处理当前项目中已有的中文乱码文本，除非它位于本次修改的样式注释或主题设置显示文本中。

## 验收标准

- 默认打开系统后，整体是浅色后台风格。
- 侧栏、顶部栏、内容区、表格、弹窗视觉统一。
- 主要操作按钮、菜单选中态、输入框焦点态均使用同一青色主色。
- 表格页面仍保持高信息密度，不被大卡片或大动效干扰。
- 首页不再像营销展示页，和后台页面气质一致。
- 移动宽度下不出现明显文字溢出或布局覆盖。

## 验证方式

- 运行 `cd frontend && npm run build`。
- 如 lint 环境可用，运行 `cd frontend && npm run lint`。
- 手动检查最小路径：
  - 登录页进入后的默认布局。
  - 首页 dashboard。
  - 用户管理表格页。
  - 菜单或部门树页面。
  - 主题设置抽屉。
  - 任意新增或编辑弹窗。

## 自检

- 范围聚焦在前端视觉，不涉及业务逻辑。
- 没有待定项。
- 没有改变路由、接口、权限或数据库。
- 设计和“浅色后台管理项目整体风格”目标一致。
