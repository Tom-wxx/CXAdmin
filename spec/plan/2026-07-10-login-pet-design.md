# 登录页动态宠物设计说明

## 目标

将登录页左侧 `CXAdmin` 标题上方现有的 64px 平台图标替换为 108px 动态宠物，并满足以下要求：

- 内置猫咪、小狗、猫头鹰三种宠物。
- 默认显示猫咪。
- 宠物眼睛、头部方向和专属行为响应鼠标方向。
- 管理员可在右上角“主题设置”抽屉中切换全局宠物。
- 只有拥有 `system:config:edit` 权限的用户能够看到和修改宠物配置。
- 配置读取异常、配置缺失或值非法时，登录页稳定回退到猫咪。
- 实现需要可维护、可扩展、可测试，不引入不必要的运行时依赖。

## 当前实现审计

### 登录页

- 登录页位于 `frontend/src/views/login/index.vue`。
- 左侧品牌区当前使用 Element Plus 的 `Platform` 图标，尺寸为 64px。
- 左侧背景已经包含青色渐变和漂浮光斑，适合保留并承载动态宠物。
- 当视口宽度不超过 960px 时，左侧区域会隐藏，右侧登录表单占满宽度。

### 主题设置

- 主题设置抽屉位于 `frontend/src/components/ThemeSettings/index.vue`。
- 当前抽屉包含布局模式、侧栏配色和自定义颜色设置。
- 当前主题选项使用 `localStorage` 保存，属于浏览器本地偏好；登录宠物是全局设置，不能复用本地持久化语义。

### 参数配置

- 后端已有 `sys_config` 参数表以及参数查询、增删改服务。
- `system:config:edit` 已用于参数修改权限控制。
- 登录页未登录即可访问，因此宠物读取需要专用公开接口；不能向未认证用户开放任意配置键查询能力。

### 测试现状

- 后端已有 JUnit 5 测试体系。
- 前端暂未配置通用单元测试框架，仅有主题验证脚本。
- 本功能包含方向计算和组件生命周期行为，需要增加可重复执行的前端自动化测试。

## 方案选择

### 采用方案：内联 SVG Vue 组件

三只宠物分别使用内联 SVG Vue 组件实现，外层由统一的 `LoginPet` 组件负责类型选择、鼠标追踪和运动参数分发。

选择原因：

- SVG 在 108px 尺寸和高分辨率屏幕上保持清晰。
- 眼睛、头部、耳朵、尾巴、舌头和翅膀可以作为独立分组变换。
- 无需图片请求、Canvas 绘制或第三方动画运行时。
- 每只宠物可以独立维护，也可按相同接口继续增加新类型。
- 方向计算可从视觉渲染中抽离并单独测试。

### 备选方案：Lottie

Lottie 仅作为未来视觉复杂度显著提升时的备选方案。本次不安装 Lottie 依赖、不增加动画 JSON 资源，也不实现运行时双轨切换。

### 未采用方案：HTML 与 CSS 拼图

纯 HTML 和 CSS 能完成基础造型，但需要较多装饰节点，复杂部位变换容易与 DOM 结构耦合，后续维护和扩展成本高于 SVG。

## 组件设计

### `LoginPet`

统一入口组件，职责如下：

- 接收 `type: 'cat' | 'dog' | 'owl'`。
- 接收尺寸参数，登录页固定传入 `108`。
- 监听页面级指针移动，并生成标准化运动状态。
- 使用 `requestAnimationFrame` 合并高频事件。
- 根据宠物类型渲染对应 SVG 子组件。
- 在组件卸载时取消动画帧并清理事件监听器。
- 在鼠标离开页面后，将运动状态平滑恢复到正面。
- 响应 `prefers-reduced-motion`，降低或关闭非必要动画。

### SVG 子组件

三个子组件使用一致的输入协议，只负责渲染：

- `CatPet`：眼睛跟随、头部轻转、耳朵响应上下方向、尾巴低频摆动。
- `DogPet`：眼睛和头部跟随、耳朵轻摆、尾巴或舌头低频待机。
- `OwlPet`：大眼睛跟随、头部转向幅度略大、翅膀轻微响应方向。

子组件不直接监听浏览器事件，避免重复监听并保持可测试性。

### 方向计算工具

独立纯函数负责将鼠标位置和宠物边界转换为运动状态：

- 计算鼠标相对宠物中心的横向、纵向方向。
- 将方向标准化到 `-1` 至 `1`。
- 对眼睛位移、头部平移和旋转分别设置上限。
- 处理鼠标与宠物中心重合、边界尺寸为零等输入。
- 输出只包含数值，不操作 DOM。

纯函数边界使后续调整动作幅度或增加新宠物时不影响事件层和渲染层。

## 数据模型与接口

### 数据库存储

在 `sys_config` 中增加一个系统内置配置：

- 配置名称：登录页宠物
- 配置键：`sys.login.pet.type`
- 默认值：`cat`
- 系统内置：`Y`
- 允许值：`cat`、`dog`、`owl`

数据库初始化脚本提供默认记录。系统内置配置不能通过现有删除流程删除。

### 公开读取接口

新增 `GET /system/config/public/login-pet` 专用只读接口。接口只返回登录宠物类型，不接受任意配置键参数，并在 Spring Security 中将该固定路径加入匿名访问白名单。

成功响应的 `data` 字段直接返回 `cat`、`dog` 或 `owl` 字符串，沿用项目现有统一 `Result<String>` 响应结构。

返回规则：

- 数据库值为 `cat`、`dog` 或 `owl` 时返回该值。
- 配置不存在、值为空或值非法时返回 `cat`。
- 数据库读取异常由前端继续使用本地默认值 `cat`，登录功能不因外观配置失败而中断。

### 管理员更新接口

新增 `PUT /system/config/login-pet` 专用更新接口，并使用 `@PreAuthorize("@ss.hasPermi('system:config:edit')")` 限制访问。请求体固定为 `{ "type": "cat" }` 结构，`type` 可替换为三个白名单值之一；响应沿用项目统一 `Result<Void>` 结构。

更新规则：

- 请求值必须是 `cat`、`dog` 或 `owl`。
- 非法值返回参数错误，不写入数据库。
- 更新操作沿用参数配置服务并启用事务。
- 普通用户即使直接调用接口也会被拒绝。

### 前端数据流

1. 登录页初始化时默认设置为 `cat`。
2. 登录页异步调用公开读取接口。
3. 返回合法值后切换对应宠物；请求失败时继续显示猫咪。
4. 管理员打开主题设置抽屉时读取当前全局宠物。
5. 管理员点击一个宠物缩略图后调用更新接口。
6. 保存成功后更新抽屉内选中态和预览，并显示成功提示。
7. 保存失败时恢复原选中项并显示错误提示。
8. 所有用户下次进入或刷新登录页时读取最新全局配置。

## 主题设置抽屉

新增“登录页宠物”设置区块：

- 仅在用户拥有 `system:config:edit` 权限时渲染。
- 横向展示猫咪、小狗、猫头鹰三个可点击缩略图。
- 每个选项显示宠物缩略图和名称。
- 当前选项使用青色边框、勾选标记和清晰文字状态。
- 保存期间禁用重复点击，避免并发更新。
- 选项使用可聚焦的语义化按钮，支持键盘操作和可见焦点状态。
- 缩略图不运行完整页面级鼠标监听，仅展示轻量待机效果或静态姿态。

主题设置中的本地侧栏颜色和布局偏好保持现状；登录宠物通过后端保存，不写入 `localStorage`。

## 登录页视觉

- 使用 108px 动态宠物替换原 64px 平台图标。
- 宠物继续位于 `CXAdmin` 标题上方。
- 保留当前青色渐变背景、漂浮光斑、标题和说明文字。
- 调整宠物与标题之间的间距，使宠物清晰可见但不压过品牌标题。
- SVG 主色与现有青色品牌保持协调，小狗允许使用受控暖色作为角色识别色。
- 宠物容器不拦截指针事件，不影响登录表单或页面操作。
- 960px 以下继续隐藏整个左侧区域，不改变移动端表单布局。

## 动效规范

### 页面级响应

- 鼠标在整个登录页移动时，宠物朝向鼠标。
- 眼睛位移、头部平移和旋转均设置固定上限。
- 动画只更新 `transform` 或 SVG 变换属性，避免触发布局重排。
- `requestAnimationFrame` 中只应用最新一次指针位置。
- 鼠标离开页面时，宠物平滑回到正面。

### 待机行为

- 三只宠物均有低频眨眼。
- 猫咪尾巴轻微摆动。
- 小狗尾巴或舌头进行低频友好动作。
- 猫头鹰翅膀进行轻微呼吸式动作。
- 待机动作保持克制，不与登录表单争夺注意力。

### 减少动态效果

当系统启用 `prefers-reduced-motion: reduce` 时：

- 关闭尾巴、翅膀、舌头和眨眼等持续动画。
- 保留幅度更小的眼睛和方向反馈。
- 移除不必要的缓动和回弹。

## 错误与边界处理

- 公开配置读取失败：保留默认猫咪，不阻塞登录页。
- 返回值非法：前后端均归一化为猫咪或拒绝保存。
- 管理员保存失败：恢复原选项并显示错误提示。
- 快速连续选择：保存期间禁用选项，避免竞态覆盖。
- 组件卸载：移除指针和媒体查询监听器，取消未执行动画帧。
- 页面不可见：停止不必要的持续计算，重新可见后恢复。
- 不支持精细指针的设备：保持正面姿态和低频待机动画。

## 安全设计

- 公开接口只暴露单一白名单枚举值，不提供任意系统参数读取能力。
- 更新接口在后端强制校验 `system:config:edit`，不能只依赖前端隐藏。
- 服务层再次验证宠物类型，防止绕过前端提交非法值。
- SVG 为本地静态模板，不渲染来自数据库或用户输入的 SVG 内容。

## 测试设计

### 前端自动化测试

引入 Vitest，并为以下行为增加测试：

- 相对方向标准化和边界限制。
- 鼠标位于中心、四个方向和极远位置时的计算结果。
- 零尺寸边界和异常数值处理。
- 非法宠物类型回退到猫咪。
- 正确渲染猫咪、小狗和猫头鹰。
- 指针事件通过 `requestAnimationFrame` 合并。
- 组件卸载后监听器和动画帧得到清理。
- 减少动态效果模式下关闭持续动画。
- 主题设置权限不足时不渲染宠物选择区。
- 保存成功更新选中态，保存失败恢复原值。

### 后端自动化测试

- 配置缺失时读取结果为 `cat`。
- 三个合法值可以读取和保存。
- 非法值被拒绝且数据库不发生修改。
- 公开读取接口只返回白名单值。
- 拥有 `system:config:edit` 权限的用户可以更新。
- 无权限用户调用更新接口时被拒绝。

## 验收标准

- 登录页默认显示 108px 猫咪，替换原平台图标。
- 猫咪、小狗和猫头鹰都可被全局选用。
- 三只宠物的眼睛和方向均能稳定跟随鼠标。
- 每只宠物具有可识别且克制的专属行为。
- 只有拥有 `system:config:edit` 权限的管理员能在主题设置抽屉中看到并修改宠物。
- 普通用户无法通过界面或接口修改全局宠物。
- 更新后刷新登录页可看到新宠物。
- 配置缺失、非法或读取失败时始终回退猫咪，登录流程不受影响。
- 动画不造成明显布局抖动、输入延迟或监听器泄漏。
- 前端测试、类型检查、Lint、构建及相关后端测试通过。

## 本次不包含

- 不在注册、忘记密码和重置密码页面增加动态宠物。
- 不改变登录表单、认证流程、SSO、验证码或路由。
- 不改变 960px 以下隐藏左侧区域的现有响应式策略。
- 不实现按用户保存不同宠物。
- 不引入 Lottie、Canvas 动画框架或远程宠物资源。
- 不提供宠物颜色、动作强度、尺寸或自定义上传配置。
